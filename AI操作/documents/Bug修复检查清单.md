## 🔍 常见Bug模式

### A. 常见错误类型
1. **TypeError** - 类型错误
2. **ReferenceError** - 引用错误
3. **SyntaxError** - 语法错误
4. **RangeError** - 范围错误
5. **DOMException** - DOM异常

### B. 容易出错的地方
- `localStorage.getItem()` 返回 `null`
- `document.getElementById()` 返回 `null`
- 数组索引访问
- 对象属性访问
- 函数参数数量不匹配
- 异步操作竞态条件

## 🛠️ 调试和测试方法

### 1. 错误捕获
```javascript
try {
    // 可能出错的代码
} catch (error) {
    console.error('错误:', error);
    // 错误处理
}
```

### 2. 类型检查
```javascript
function safeDivide(a, b) {
    if (typeof a !== 'number' || typeof b !== 'number') {
        return 0;
    }
    if (b === 0 || !isFinite(b)) {
        return 0;
    }
    return a / b;
}
```

### 3. 存在性检查
```javascript
if (element && element.classList) {
    element.classList.add('active');
}
```

### 4. 安全函数
- `safeDivide(num, denom)` - 安全除法
- `safeParseNumber(value, defaultValue)` - 安全数字解析
- `safeMax(arr)` - 安全求最大值
- `safeMin(arr)` - 安全求最小值

## 🔧 安全修复实践

1. **最小修改原则** - 只修改必要的代码
2. **测试覆盖** - 修复后进行完整测试
3. **回归测试** - 确保不破坏其他功能
4. **代码审查** - 修复后检查代码质量
