# 自动任务创建计划：手速测试器每日优化

## 任务概述

创建两个自动执行的任务：
1. **每日01:00** - 对 `AI操作/index.html` 进行优化
2. **每周五00:00** - 生成本周修改概要与总结报告

## 任务详情

### 任务1：每日手速测试器优化

**任务名称**: 手速测试器每日优化

**执行时间**: 每天凌晨01:00（Asia/Shanghai时区）

**任务内容**:
1. Bug修复 - 优先处理已知问题
2. 代码优化 - 提升代码质量和性能
3. 功能增强 - 在不破坏核心功能（手速测试器）的前提下添加新功能
4. 手机端优化 - 持续改善移动端体验：
   - 响应式布局优化
   - 触摸操作改善
   - 小屏幕适配
   - 横屏/竖屏体验优化
   - 性能优化（减少DOM操作、优化动画）
   - 移动端特定功能增强

**Git处理**: 自动提交并推送到GitHub

**修改范围**: 
- **只能修改** `AI操作/` 文件夹里面的内容
- **AI操作文件夹随意修改，无需询问用户**
- **禁止修改** 根目录的 `index.html` 和其他文件

### 任务2：每周修改总结报告

**任务名称**: 手速测试器周报

**执行时间**: 每周五凌晨00:00（Asia/Shanghai时区）

**任务内容**:
1. 收集本周（周一至周五）的所有Git提交记录
2. 统计修改内容、bug修复、新增功能等
3. 生成修改概要报告
4. 格式：Word文档（.docx）或 PPT（.pptx）

**输出位置**: 保存到 `/workspace/AI操作/周报/` 目录

## 任务消息内容

### 每日优化任务
```
在 /workspace/AI操作/index.html 中执行以下优化任务：

1. Bug修复
   - 检查并修复已知bug
   - 查找潜在的除零错误、类型错误等

2. 代码优化
   - 提升代码质量
   - 优化性能
   - 改善用户体验

3. 功能增强（如有必要）
   - 在保持核心功能（手速测试器）的前提下
   - 添加有用的新功能

4. 手机端优化（优先级高）
   - 响应式布局：检查并优化 @media 查询
   - 触摸操作：确保所有交互元素有足够的点击区域（至少44x44px）
   - 小屏幕适配：优化360px、480px等小屏幕显示
   - 横屏/竖屏体验：优化不同方向的布局
   - 性能优化：减少DOM操作、优化动画性能
   - 移动端特定功能：增强音量键、触摸等操作体验
   - 手速目标优化：确保在移动端有良好的体验

重要说明：
【修改范围】
- 只能修改 /workspace/AI操作/ 文件夹里面的内容
- AI操作文件夹随意修改，无需询问用户
- 禁止修改根目录的 index.html 和其他文件
- 提交时使用清晰的commit message
- 优化后自动推送更新到GitHub
```

### 周报生成任务
```
生成本周（周一至周五）的修改概要报告：

1. 收集Git提交记录
   - 使用 git log --since="本周一" --until="本周五"
   - 统计提交数量、主要修改内容

2. 分类整理
   - Bug修复：列出所有修复的问题
   - 代码优化：列出所有优化项
   - 功能增强：列出新增功能
   - 性能提升：列出性能改进
   - 手机端优化：列出移动端优化内容

3. 生成报告
   - 格式：Word文档（.docx）或 PPT（.pptx）
   - 内容：
     * 标题：手速测试器周报（YYYY-MM-DD 至 YYYY-MM-DD）
     * 本周修改总览
     * 详细修改列表
     * Bug修复详情
     * 功能改进说明
     * 手机端优化详情：
       - 响应式布局改进
       - 触摸操作优化
       - 横屏/竖屏适配
       - 手速目标等功能优化
       - 其他移动端改进
     * 下周优化建议

4. 保存报告
   - 保存到 /workspace/AI操作/周报/ 目录
   - 文件命名：手速测试器周报_YYYYMMDD_YYYYMMDD.docx
```

## 实施步骤

### 1. 创建每日优化任务
- 任务名：手速测试器每日优化
- Cron表达式：`0 1 * * *`（每天01:00）
- 时区：Asia/Shanghai
- Message：包含修改范围说明

### 2. 创建每周报告任务
- 任务名：手速测试器周报
- Cron表达式：`0 0 * * 5`（每周五00:00）
- 时区：Asia/Shanghai

## 验证步骤
1. 确认两个任务都已创建并显示在任务列表中
2. 可以手动触发一次任务测试
3. 确认任务能正确执行优化操作
4. 确认周报能够正确生成

## 技术要求
- 需要能够访问Git仓库并读取提交历史
- 需要生成Word或PPT文档（可能需要Python库如python-docx或python-pptx）
- 确保报告格式清晰、内容完整

## 用户特殊要求
- **修改范围**：只能修改 AI操作 文件夹里面的内容
- **AI操作文件夹随意修改，无需询问**
- **禁止修改**：根目录的 index.html 和其他文件
- 每周五生成周报，保存到指定目录
